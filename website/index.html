  • Sat Nov 29 2003: OpenWBEM 2.0.14 has been released.
    This release fixes a few bugs since 2.0.8.

    Changes:
    • Fixed the index code to flush to disk after a modifying operation in order to prevent database corruption if the cimom isn't shut down properly.
    • Removed a couple of extra CRLFs that were being generated by the HTTP client.
    • Fixed the spec file to build on Fedora Core 1
    • Added locking in provider unloading to avoid a race condition.
    • Fixed OW_Array::operator+= to return *this
    • Fixed an issue with the server generating non-standard xml when doing an AssociatorNames operation on class paths.
    • Fixed a bug where a NULL CLASSNAME arg wasn't handled
    • Fixed EnumerateClassNames to be about 10x faster
    • Changed createInstance to pass a class-synced-up instance to the provider instead of the raw instance that comes in from the client.
    • Fixed a problem where the instance keys may not be correctly set in the client API when calling associators or references.
    • Fixed an issue with CreateInstance when the new instance didn't have any KEY qualifiers.

  • Sun Oct 12 2003: OpenWBEM 2.0.8 has been released.
    This release fixed a few bugs in 2.0.6.

    Changes:
    • Added support for compiling on OS X(Darwin). Note that the code will compile but the current Makefile build system doesn't work yet.
    • Fixed possible segfault if an auth module failed to initialize
    • Fixed problems with RedHat 9
    • Fixed a problem with getInstance on association instances without namespaces on the reference properties
    • Fixed slp provider to register when the http ports aren't specified in the config file
    • Added workaround for clients who send invalid xml as the namespace
    • Fix for deflate compression

  • Mon May 19 2003: OpenWBEM 2.0.6 has been released.
    This release fixed a few bugs in 2.0.4. There are no new features. Note that you will have to re-create the CIM repositories (found in var/openwbem), because a fix for the association handling required a new data format.

    Changes:
    • Fixed some problems with the association database index.
    • Fixed a problem in OW_CIMServer that caused associations to be broken in certain cases.
    • Removed _validatePropagatedKeys from OW_CIMServer because it doesn't work right.
    • Fixed a couple of problems with xml generation where an object path wasn't being done correctly.
    • Fixed the simple c++ provider interface name.

The OpenWBEM project is an open-source implementation of Web Based Enterprise Management suitable for commercial and non-commercial applications.

Some of the main features of OpenWBEM are:
  • OpenWBEM CIMOM
    • CIM Operations over HTTP
    • CIM Operations over HTTPS (Requires OpenSSL)
    • Extensible Authentication through authentication plug-in modules
      • PAM Authentication Module
      • SIMPLE Authentication Module
      • Digest Authentication Module
    • Fast scalable hierarchical repository with indexing
    • Extensible Provider Interface through Provider Interface plug-ins
      • C++ Provider Interface
      • NPI Provider Interface. Thanks to IBM!
    • Providers:
      • Instance
      • Method
      • Associator
      • Property
      • Indication Export
      • Polled (can be used as an Indication provider)
      • Authentication
      • WQL
      • Writing providers is easy. Providers don't need to worry about locking, since the cimom takes care of it.
    • Indications and CIM Listener support
    • WQL Level 2 support for server and client
    • Access Control Lists
    • SLP Integration on the CIMOM and client side using any RFC 2614 compliant SLP implementation (e.g. OpenSLP)
    • Support for a binary protocol that is much more efficient than CIM/XML
    • http compression. (Using the binary protocol and compression is very bandwidth efficient.)
    • http chunking allows openwbem to transfer large transactions (e.g. enumClasses on the entire schema) with no buffering. Memory usage stays constant.
    • Communication over TCP/IP or Domain sockets.
    • The owcimomd daemon has been designed to be non-intrusive and have a small footprint. Many features are loaded as shared libraries so you can pick and choose what features you really need and not waste memory with unused code.
      • Indication support
      • WQL library
      • Authentication modules
      • Provider interfaces
      • Providers are unloaded from memory if they are not used for a configurable amount of time
      • SLP support is implemented as a provider
      • Request handlers (CIM/XML or binary) can be loaded/unloaded on demand with a configurable unload timeout
  • Additional Software
    • C++ Client API library
    • MOF Compiler w/embeddable library
    • WQL command-line utility

Download

Please read the license before you download. The current release can be downloaded from SourceForge.

The most recent development code can be obtained from cvs.openwbem.org.  You can obtain the code by using either cvs pserver or downloading a tarball via viewcvs.
A note about the development code: We strive to keep the code bug-free and stable.  However, there may be experimental features, unstable APIs, or even (heaven forbid!) code that doesn't compile.  So if you can't deal with these slight annoyances (believe me, we don't break things on purpose :-), please use the latest released version of OpenWBEM.

Related Projects

  • Owperlprovider (http://jason.long.name/owperl/) - "a new Perl provider interface for OpenWBEM. This lets you write OpenWBEM providers in Perl."
  • kim-browser (http://sourceforge.net/projects/kde-cim) - "The KDE CIM Browser is a KDE based frontend to management brokers using the CIM-XML protocol. The tool will be useful to view and modify CIM class hierarchies as well as instance and association structures."  Check out the latest code from CVS for better functionality.
  • CimNavigator (http://cimnavigator.com/) - "a Java client application that can be used to graphically browse CIM objects and their associations in various CIMOMs."
  • SBLIM (http://www-124.ibm.com/sblim/) - A lot of their Linux providers work with OpenWBEM NPI & CMPI provider interfaces.  "SBLIM (pronounced "sublime"), the Standards Based Linux Instrumentation for Manageability is an IBM Open Source project, intended to enhance the manageability of GNU/Linux systems."
  • WBEM Services (http://wbemservices.sourceforge.net/) - "The WBEM Services project is an effort to develop an open-source Java™ implementation of Web Based Enterprise Management (WBEM) suitable for commercial and non-commercial application. This project consists of APIs, server and client applications and tools."  I find their browser to be useful.
  • OpenPegasus (http://openpegasus.org/) - "Pegasus is an open-source implementation of the DMTF CIM and WBEM standards."  They have a good Java browser that used to be known as the SNIA CIM Browser.
